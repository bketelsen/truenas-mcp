# Epic: Safe read-only operation mode

## Purpose

Users can run the MCP server in a mode that guarantees no destructive or state-changing operations are possible, so that AI assistants can safely query the NAS without risk of accidental data loss or configuration changes.

---

## Scope

- A CLI flag that restricts the MCP server to read-only tools only
- When enabled, tools that create, delete, update, dismiss, start, stop, or restart resources are not exposed to AI clients
- Read-only tools (list, get, inspect) remain fully functional

---

## Out of Scope

- Per-tool granular allow/deny lists
- Role-based access control
- Runtime toggling (flag is set at startup only)

---

## Stories

> Stories are tracked in the backlog. This list is descriptive only.

- E07S01: Read-only flag prevents destructive tool registration

---

## Success Metrics

- When `--read-only` is passed, no destructive tool appears in the MCP tool listing
- When `--read-only` is omitted, all tools are available (backwards compatible)

---

## Dependencies

- E01 (foundation and MCP server initialization must exist)
