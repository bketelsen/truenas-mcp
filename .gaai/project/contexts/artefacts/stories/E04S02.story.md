# Story: Create, update, and delete shares

## User Story

**As an** AI assistant user,
**I want** to ask the AI to create, modify, or remove network shares,
**so that** I can manage file sharing without accessing the TrueNAS UI.

---

## Context

Exposes `sharing.smb.create/update/delete` and `sharing.nfs.create/update/delete` via MCP tools.

---

## Acceptance Criteria

- [ ] AC1: `truenas_smb_create` tool creates an SMB share with required `name` and `path`, optional `comment` and `guest_ok`
- [ ] AC2: `truenas_smb_delete` tool deletes an SMB share by ID
- [ ] AC3: `truenas_nfs_create` tool creates an NFS export with required `path` and optional `networks` and `hosts`
- [ ] AC4: `truenas_nfs_delete` tool deletes an NFS export by ID
- [ ] AC5: All tools return clear success/failure messages

---

## Out of Scope

- Service start/stop (SMB/NFS services)
- ACL or permission management

---

## Dependencies

- E04S01 (share visibility for verification)

---

## Notes

Share creation requires the underlying dataset/path to exist. Tool descriptions should note this.
