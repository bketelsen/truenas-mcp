# Story: Project scaffolding with fang/cobra and serve command

## User Story

**As an** AI assistant user,
**I want** a `truenas-mcp serve` command that starts the MCP server,
**so that** I can configure my AI tool to launch it as an MCP server process.

---

## Context

This is the project bootstrap — Go module, CLI entry point, and the `serve` command that will host the MCP server. Uses charmbracelet/fang v2 wrapping cobra for polished CLI output.

---

## Acceptance Criteria

- [ ] AC1: Go module is initialized with `go.mod` including fang v2, cobra, and go-sdk dependencies
- [ ] AC2: `main.go` uses `fang.Execute` to run a cobra root command with app name and description
- [ ] AC3: A `serve` subcommand exists that accepts `--host` and `--api-key` flags (or env vars `TRUENAS_HOST`, `TRUENAS_API_KEY`)
- [ ] AC4: Running `truenas-mcp serve` without required config prints a clear error message
- [ ] AC5: `truenas-mcp --version` outputs the version string

---

## Out of Scope

- Actual MCP server logic (E01S03)
- TrueNAS connection (E01S02)

---

## Dependencies

None

---

## Notes

Config should support both flags and environment variables. Env vars take precedence for secret values (API key).
