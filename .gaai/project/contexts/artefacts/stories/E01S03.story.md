# Story: MCP server initialization with stdio transport

## User Story

**As an** AI assistant user,
**I want** the `serve` command to run a working MCP server over stdio,
**so that** my AI tool can discover available TrueNAS tools.

---

## Context

The MCP server uses the official `modelcontextprotocol/go-sdk` with stdio transport. On startup it should connect to TrueNAS, register tools, and begin serving MCP requests. A `system.info` ping tool serves as the first registered tool to verify end-to-end connectivity.

---

## Acceptance Criteria

- [ ] AC1: `truenas-mcp serve` starts an MCP server listening on stdio
- [ ] AC2: Server responds to MCP `initialize` handshake with server name and version
- [ ] AC3: Server responds to `tools/list` with at least one registered tool (`truenas_system_info`)
- [ ] AC4: The `truenas_system_info` tool calls `system.info` on TrueNAS and returns the result
- [ ] AC5: Server shuts down cleanly on SIGINT/SIGTERM

---

## Out of Scope

- Domain-specific tools beyond `system.info` (handled by E02–E06)

---

## Dependencies

- E01S01 (CLI scaffolding)
- E01S02 (TrueNAS connection)
