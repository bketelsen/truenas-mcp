# Story: TrueNAS WebSocket connection and authentication

## User Story

**As an** AI assistant user,
**I want** the MCP server to connect and authenticate to my TrueNAS server,
**so that** it can make API calls on my behalf.

---

## Context

TrueNAS SCALE Goldeye uses WebSocket JSON-RPC 2.0 at `wss://<host>:444/api/current`. Authentication is via `auth.login_with_api_key`. Uses the official `truenas/api_client_golang` library.

---

## Acceptance Criteria

- [ ] AC1: Server establishes a WebSocket connection to the configured TrueNAS host
- [ ] AC2: Server authenticates using the provided API key via `auth.login_with_api_key`
- [ ] AC3: Authentication failure produces a clear error message and exits gracefully
- [ ] AC4: Connection loss is detected and reported (no silent failures)

---

## Out of Scope

- Automatic reconnection / retry logic
- Connection pooling

---

## Dependencies

- E01S01 (CLI scaffolding with config)

---

## Notes

The official Go client handles WebSocket framing. Self-signed TLS certificates on the NAS may require a flag to skip verification.
