# Story: System information and disk health

## User Story

**As an** AI assistant user,
**I want** to ask the AI about my NAS system status and disk health,
**so that** I can monitor my server without opening the TrueNAS UI.

---

## Context

Exposes `system.info`, `disk.query`, and related methods via MCP tools.

---

## Acceptance Criteria

- [ ] AC1: A `truenas_system_info` tool returns hostname, version, uptime, and platform info
- [ ] AC2: A `truenas_disk_list` tool returns all disks with name, size, model, serial, and health status
- [ ] AC3: A `truenas_network_list` tool returns network interfaces with IP addresses and link status

---

## Out of Scope

- System configuration changes
- Reboot/shutdown commands
- Update management

---

## Dependencies

- E01S03 (MCP server with TrueNAS connection)

---

## Notes

`system.info` is already used as the ping tool in E01S03. This story expands it with disk and network visibility.
