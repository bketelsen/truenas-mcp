# Story: List and inspect snapshots

## User Story

**As an** AI assistant user,
**I want** to ask the AI to list snapshots for a dataset,
**so that** I can review backup points and storage usage.

---

## Context

Exposes `zfs.snapshot.query` via MCP tools. Snapshots are key to data protection in ZFS. Note: snapshot endpoints are WebSocket-only in Goldeye (no REST fallback).

---

## Acceptance Criteria

- [ ] AC1: A `truenas_snapshot_list` MCP tool exists that returns snapshots with name, dataset, creation time, and referenced size
- [ ] AC2: Tool supports a required `dataset` parameter to scope results
- [ ] AC3: A `truenas_snapshot_get` MCP tool exists that returns full details for a specific snapshot by name

---

## Out of Scope

- Snapshot creation or deletion (E03S02)
- Replication

---

## Dependencies

- E01S03 (MCP server with TrueNAS connection)

---

## Notes

Snapshot lists can be very large. Always require dataset scoping.
