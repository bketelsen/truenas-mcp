# Story: Create and delete snapshots

## User Story

**As an** AI assistant user,
**I want** to ask the AI to create or delete snapshots,
**so that** I can manage backup points and clean up old snapshots through conversation.

---

## Context

Exposes `zfs.snapshot.create` and `zfs.snapshot.delete` via MCP tools. Snapshot creation is safe; deletion is destructive but recoverable at the dataset level.

---

## Acceptance Criteria

- [ ] AC1: A `truenas_snapshot_create` MCP tool exists that creates a snapshot with required `dataset` and optional `name` (auto-generates timestamp-based name if omitted)
- [ ] AC2: A `truenas_snapshot_delete` MCP tool exists that deletes a snapshot by full name
- [ ] AC3: Both tools return clear success/failure messages with the snapshot name

---

## Out of Scope

- Snapshot rollback
- Snapshot cloning
- Bulk snapshot cleanup policies

---

## Dependencies

- E02S03 (snapshot visibility for verification)

---

## Notes

Snapshot names follow the convention `dataset@snapname`. The create tool should handle this formatting.
