# Story: Create and delete datasets

## User Story

**As an** AI assistant user,
**I want** to ask the AI to create or delete datasets on my NAS,
**so that** I can manage storage organization through conversation.

---

## Context

Exposes `pool.dataset.create` and `pool.dataset.delete` via MCP tools. These are mutating operations — the AI assistant should confirm before executing.

---

## Acceptance Criteria

- [ ] AC1: A `truenas_dataset_create` MCP tool exists that creates a dataset with required `name` (full path) and optional `comments`, `compression`, `quota` parameters
- [ ] AC2: A `truenas_dataset_delete` MCP tool exists that deletes a dataset by path
- [ ] AC3: Both tools return clear success/failure messages
- [ ] AC4: Delete tool requires explicit dataset path (no wildcards or recursive delete by default)

---

## Out of Scope

- Dataset property modification beyond creation parameters
- Recursive dataset operations

---

## Dependencies

- E02S02 (dataset visibility for verification)

---

## Notes

Deletion is destructive. The MCP tool description should clearly state this so AI assistants can warn users.
